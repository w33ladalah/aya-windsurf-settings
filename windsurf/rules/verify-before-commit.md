# Verify Before Commit

## Rule

Before committing any bug fixes or new feature implementations, **always ask the user to verify** that the bug has been fixed or the feature is working as expected.

## When to Apply

- After fixing a bug (backend, frontend, or admin)
- After implementing a new feature
- After making any code changes that affect functionality

## What to Ask

Before suggesting a git commit, ask:

- "Has the bug been fixed and is it working as expected?"
- "Is the new feature working as expected?"

## Why

This ensures that changes are tested and verified before being committed to version control, preventing unverified code from being added to the repository.
